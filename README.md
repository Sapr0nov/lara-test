 # Тестовое Задание

 ## Описание: 
необходимо написать код, по которому происходит зачисление и списание крипто-баланса пользователя, с учетом рисков

 ## Цель:
 - Разработать модуль учета крипто-баланса пользователя, который позволяет корректно и безопасно:
зачислять средства
 - списывать средства (выводы, платежи, комиссии),
с учетом специфики блокчейна, асинхронности и рисков

 ## Ограничения и допущения
 - Используйте Стек технологий PHP+Laravel
 - Предполагаем что класс пользователя уже существуют (User)
 - У пользователя есть класс Кошелёк для каждой монеты (Wallet реализация вне тестового) 
 - В системе есть Enum, .env или класс с параметрами по каждой монете (например, CoinParams) где хранится количество подтвержденных блоков транзакции, чтобы считать  платеж успешным

 # Шаги выполнение
 - создали репозиторий и развернули стандартный ларавел 
 - уточнили специфику бокчейна (смотрим: https://developer.bitcoin.org/devguide/payment_processing.html и https://ethereum.org/ru/developers/tutorials/learn-foundational-ethereum-topics-with-sql/)
 - изучили доступный код в документации https://doc-merchant.alikassa.io/ru/other/full-code/

 ## Специфика блокчейна
 - распределенная система, требующая подтверждения несколькими блоками (confirmations)
 - разные монеты используют разные подоходы, например, модель account based (Ethereum) или модель UTXO (Bitcoin)
 - блокчейн это не баланс аккаунта, а цепочка операций

 ## Выводы по архитектуре
 - один модуль, но с разными драйверами (реализацию драйверов вынесем за тестовое задание :) )
 - Модель хранения - не баланс а история операций (ledger)
 - Фоновая обработка подтверждений
 - использование пхп библиотеки ext-bcmath

### Фоновые задачи
Операции блокчейна асинхронны. После создания операции система ожидает подтверждения блоками сети.
Вариант решения - Проверка выполняется фоновыми воркерами через очередь сообщений.
